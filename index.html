Logistics Performance Analysis Window Functions

Business Problem

Logistics company experiencing rising delivery delays, SLA breaches, and inconsistent driver performance across regions, leading to customer dissatisfaction and operational inefficiencies.

SQL Solution Logic

-- Driver Performance with Rolling Trends
WITH driver_weekly AS (
  SELECT 
    driver_id,
    DATE_TRUNC('week', delivery_date) AS week,
    ROUND(100.0 * COUNT(CASE WHEN delivery_time_min > sla_time_min 
                         THEN 1 END) / COUNT(*), 2) AS delay_percent
  FROM delivery_orders 
  GROUP BY driver_id, DATE_TRUNC('week', delivery_date)
),
driver_trends AS (
  SELECT *,
    AVG(delay_percent) OVER (
      PARTITION BY driver_id 
      ORDER BY week 
      ROWS BETWEEN 2 PRECEDING AND CURRENT ROW
    ) AS rolling_3week_avg,
    RANK() OVER (ORDER BY delay_percent DESC) AS delay_rank
  FROM driver_weekly
)
SELECT * FROM driver_trends WHERE delay_percent > 25;

Key Insights

  • 33.1% delay rate for worst-performing driver on Route RTE05
  • 57.2% vehicle delay rate for specific vehicles consistently involved in delays
  • Week-over-week trends identified drivers with deteriorating performance

Business Decisions

  • Reassign high-risk driver-route combinations
  • Replace underperforming vehicles on critical routes
  • Implement targeted coaching for drivers with >25% delay rates
  • Dynamic route assignment based on historical performance data

Supply Chain Supplier Analysis Risk Assessment

Business Problem

Procurement team struggling with supplier reliability issues, inconsistent delivery performance, and lack of visibility into cost vs. quality trade-offs across vendor portfolio.

SQL Solution Logic

-- Supplier Risk Scorecard with Multi-Dimension Analysis
WITH supplier_metrics AS (
  SELECT 
    supplier_id,
    COUNT(*) as total_orders,
    AVG(delivery_days) as avg_delivery_time,
    STDDEV(delivery_days) as delivery_consistency,
    AVG(quality_score) as avg_quality,
    AVG(unit_cost) as avg_cost,
    COUNT(CASE WHEN delivery_days > promised_days THEN 1 END) 
      / COUNT(*)::FLOAT as late_delivery_rate
  FROM procurement_orders 
  GROUP BY supplier_id
),
risk_scoring AS (
  SELECT *,
    CASE 
      WHEN late_delivery_rate > 0.2 AND avg_quality < 7 THEN 'HIGH_RISK'
      WHEN late_delivery_rate > 0.1 OR avg_quality < 8 THEN 'MEDIUM_RISK'
      ELSE 'LOW_RISK'
    END as risk_category,
    RANK() OVER (ORDER BY avg_cost ASC, avg_quality DESC) as value_rank
  FROM supplier_metrics
)
SELECT * FROM risk_scoring ORDER BY risk_category, value_rank;

Key Insights

  • 23% of suppliers classified as high-risk with >20% late delivery rates
  • Cost-quality correlation revealed optimal supplier segments
  • Delivery consistency metrics identified unreliable partners

Business Decisions

  • Diversify away from high-risk suppliers
  • Negotiate improved SLAs with medium-risk vendors
  • Increase order volume with top-performing suppliers
  • Implement supplier performance monitoring dashboard

Business Problem

Retail company facing frequent stockouts and backorders leading to lost sales, poor customer experience, and inefficient inventory allocation across product categories.

SQL Solution Logic

-- Stockout Risk Analysis with Seasonal Patterns
WITH inventory_velocity AS (
  SELECT 
    product_id,
    category,
    DATE_TRUNC('month', order_date) as month,
    SUM(quantity_ordered) as monthly_demand,
    AVG(stock_level) as avg_stock,
    COUNT(CASE WHEN stock_level = 0 THEN 1 END) as stockout_days,
    LAG(SUM(quantity_ordered)) OVER (
      PARTITION BY product_id 
      ORDER BY DATE_TRUNC('month', order_date)
    ) as prev_month_demand
  FROM inventory_orders 
  GROUP BY product_id, category, DATE_TRUNC('month', order_date)
),
risk_classification AS (
  SELECT *,
    CASE 
      WHEN stockout_days > 5 AND monthly_demand > prev_month_demand * 1.2 
        THEN 'CRITICAL'
      WHEN stockout_days > 2 OR avg_stock < monthly_demand * 0.5 
        THEN 'HIGH_RISK'
      ELSE 'NORMAL'
    END as stockout_risk,
    monthly_demand / NULLIF(avg_stock, 0) as inventory_turnover
  FROM inventory_velocity
)
SELECT category, stockout_risk, COUNT(*) as product_count,
       AVG(inventory_turnover) as avg_turnover
FROM risk_classification 
GROUP BY category, stockout_risk
ORDER BY category, stockout_risk;

Key Insights

  • 15% of products classified as critical risk with frequent stockouts
  • Seasonal demand patterns revealed optimal reorder points
  • Category-level analysis showed electronics had highest turnover

Business Decisions

  • Increase safety stock for critical-risk products
  • Adjust reorder points based on seasonal demand patterns
  • Implement automated alerts for low-stock situations
  • Optimize warehouse allocation by category performance
